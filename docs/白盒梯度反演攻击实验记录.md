# 白盒梯度反演攻击实验记录

## 1. 实验目的

为了更直接地论证“原始联邦学习并不安全”，本次不再使用弱黑盒 MIA，而是改用服务器视角的白盒梯度反演攻击。

攻击目标：

- 原始 `FedNova`（无 DP）
- `FedNova + adaptive DP`

核心问题：

- 服务器若能观察客户端上传的梯度/更新，能否直接反推出客户端本地样本？

## 2. 攻击方式

本次采用的是白盒梯度反演攻击：

- 攻击者是联邦服务器
- 攻击者已知全局模型结构与参数
- 攻击者可直接观察客户端上传的梯度
- 攻击者对单样本梯度做反演，重建输入图像

具体做法：

1. 固定同一个客户端、同一个样本
2. 在无 DP 与 adaptive DP 两种设置下分别生成“服务器可见梯度”
3. 用梯度匹配优化的方式重建原始输入
4. 比较：
   - 标签是否恢复正确
   - 重建误差 `MSE`
   - 重建质量 `PSNR`
   - 梯度匹配残差

这里的攻击比黑盒 MIA 强得多，也更符合“原始 FL 的服务器威胁模型”。

## 3. 实验设置

攻击脚本：

- `scripts/run_whitebox_attack.py`

核心攻击实现：

- `src/privacy/whitebox_attack.py`

对比配置：

- 无 DP：
  - `configs/experiments/fednova_mnist_full_nodp_mia_eval.yaml`
- adaptive DP：
  - `configs/experiments/fednova_mnist_full_adaptive_dp_riskagg_eval.yaml`

样本选择：

- 客户端：`client_id = 0`
- 客户端内样本偏移：`sample_offset = 0`
- 真实标签：`8`

反演优化参数：

- `steps = 300`
- `lr = 0.1`
- `tv_weight = 1e-4`

DP 攻击设置：

- `clip_norm = 1.0`
- `noise_multiplier = 0.08`

说明：

- 这里使用的是当前 adaptive DP 方法中的低噪声阶段设置
- 这已经是对 DP 比较“苛刻”的攻击条件

输出目录：

- `outputs/20260329_215855_whitebox_attack`

## 4. 实验结果

| 方法 | 恢复标签 | 标签是否正确 | 重建 MSE | 重建 PSNR | 最终梯度匹配损失 |
| --- | ---: | ---: | ---: | ---: | ---: |
| FedNova（无 DP） | 8 | 是 | 0.3438 | 4.6370 | 0.000446 |
| FedNova + adaptive DP | 5 | 否 | 3.7589 | -5.7506 | 0.094313 |

## 5. 结果解读

这次结果和前面的黑盒 MIA 完全不同，而且更符合我们对原始 FL 风险的预期。

### 5.1 原始 FedNova 明显存在泄露风险

无 DP 情况下：

- 标签被正确恢复为 `8`
- 梯度匹配损失极低，仅 `0.000446`
- 重建误差明显更小

这说明：

- 服务器从客户端上传的精确梯度中，确实可以恢复出与原始样本高度相关的信息
- 原始联邦学习即使不共享原始数据，也不等于天然安全

### 5.2 adaptive DP 显著削弱了白盒攻击

在 adaptive DP 下：

- 恢复标签从正确的 `8` 变成了错误的 `5`
- `MSE` 从 `0.3438` 增大到 `3.7589`
- 梯度匹配损失从 `0.000446` 增大到 `0.094313`

这说明：

- 裁剪与噪声确实有效破坏了可反演性
- 服务器虽然仍能观察到更新，但已经难以从中恢复真实样本内容

## 6. 为什么这组结果比黑盒 MIA 更重要

前面的黑盒 MIA 没打出明显差异，主要原因是：

- 攻击对象只是最终全局模型
- 攻击者能力较弱
- 模型泛化后，成员与非成员输出差异不一定明显

而白盒梯度反演直接攻击的是：

- 客户端上传给服务器的梯度/更新

这恰恰是原始联邦学习最核心的隐私暴露面。

所以如果论文要论证“原始 FL 不安全”，这组白盒攻击结果比前面的黑盒 MIA 更有说服力。

## 7. 当前结论

一句话概括本次实验：

- 原始 `FedNova` 在服务器白盒梯度反演攻击下存在明显隐私泄露风险，而加入 `adaptive DP` 后，标签恢复失败、重建误差显著增大，说明我们的方法能够有效削弱服务器侧的样本反演能力。
