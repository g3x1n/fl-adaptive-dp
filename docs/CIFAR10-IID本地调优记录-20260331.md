# CIFAR10 IID 本地调优记录（2026-03-31）

## 背景

远程 `exp_a_iid_cifar10_fedavg` 及其 DP 变体结果偏低：

- `FedAvg`: `final_test_accuracy = 0.6801`
- `FedAvg + Fixed DP`: `final_test_accuracy = 0.5363`
- `FedAvg + Adaptive DP`: `final_test_accuracy = 0.5270`

同时，`exp_a_iid_cifar10_fednova_adaptive_dp` 在远程 `CUDA` 环境报错：

- `RuntimeError: Expected all tensors to be on the same device, but found at least two devices, cuda:0 and cpu!`

## 原因分析

初步判断，`CIFAR-10 IID` 的底座偏弱不只是单个方法问题，而是共同训练配置对该任务不够友好：

- 优化器过于保守：此前仅使用 `SGD(lr)`，没有 `momentum`、`weight_decay`、`nesterov`
- `CIFAR-10` 训练无数据增强
- `CIFAR` 模型容量偏小，缺少 `BatchNorm / Dropout`
- `FedProx` 默认 `proximal_mu=0.5` 过大，容易抑制学习
