## 2. 论文实验要回答的 5 个核心问题

### 2.1 Non-IID 是否显著影响联邦学习性能

需要证明：

- 随着数据异构程度增强，训练稳定性下降
- `FedAvg` 在强 Non-IID 下更容易退化

### 2.2 异构优化算法是否能缓解 Non-IID 带来的性能退化

需要证明：

- `FedProx`、`FedNova` 相比 `FedAvg` 更稳
- 它们能提升最终精度、收敛性或抗震荡能力

### 2.3 无 DP 基线是否存在可利用的隐私泄漏风险

需要证明：

- 无 DP 的联邦学习虽然精度较高，但并不具备形式化隐私保证
- 服务器若能观察客户端上传模型或更新，就可能实施有效的隐私攻击
- 后续 DP 带来的性能下降，是“换取可验证隐私保护”的代价，而不是单纯副作用

### 2.4 固定差分隐私是否带来明显的“隐私-效能”权衡

需要证明：

- 引入固定 DP 后，模型精度通常下降
- 攻击成功率下降，但精度也下降
- 不同 `epsilon` 下存在明显权衡关系

### 2.5 自适应差分隐私是否优于固定 DP

需要证明：

- 在相同或相近隐私约束下，自适应 DP 比固定 DP 精度更高
- 或在相近精度下，自适应 DP 消耗更少隐私预算、抑制更强攻击效果
- 若结合异构优化与通信压缩，整体方案在精度、隐私、稳定性和通信开销之间更均衡

## 4. 所有实验的统一约定

## 4.1 数据集

- `MNIST`
- `CIFAR-10`

建议使用顺序：

- 前期调参与多组对比以 `MNIST` 为主
- 论文正式主结果再扩展到 `CIFAR-10`

## 4.2 数据划分

- IID：均匀划分
- Non-IID：Dirichlet 划分

建议重点取值：

- `alpha = 1.0`：轻度异构
- `alpha = 0.5`：中度异构
- `alpha = 0.1`：强异构

说明：

- 如果某个最优参数是在 `alpha = 0.1` 下调出的，正式实验可以先以 `alpha = 0.1` 为主
- 若论文正文需要更完整论证，再在实验 2 中补 `1.0 / 0.5 / 0.1`

## 4.3 训练设置

除非该实验就是在研究某个超参数，否则建议以下变量尽量固定：

- 模型结构
- `rounds`
- `local_epochs`
- `batch_size`
- `learning_rate`
- `num_clients`
- 随机种子

这样可以避免多个变量一起变动，导致实验结论模糊。

### 5.2 实验设置

### 5.3 需要输出的结果

- 每轮损失曲线

- 最佳轮次到最终轮次的回落幅度

- IID 下三种算法差异不会特别大
- Non-IID 下 `FedAvg` 更容易退化
- `FedProx`、`FedNova` 更稳健

## 6. 实验 2：Non-IID 强度影响实验

### 6.1 实验目的

验证异构程度如何影响联邦学习性能。

### 6.2 实验设置

固定算法，逐步改变：

- `alpha = 1.0`
- `alpha = 0.5`
- `alpha = 0.1`

建议算法至少包含：

- `FedAvg`
- `FedProx`
- `FedNova`

### 6.3 需要输出的结果

- `alpha - accuracy` 曲线
- `alpha - loss` 曲线
- `alpha - 收敛速度` 曲线

### 6.4 预期结论

- `alpha` 越小，性能整体越差
- `FedAvg` 对异构更敏感
- `FedProx`、`FedNova` 退化更缓

## 7. 实验 3：无隐私保护基线的安全性论证实验

### 7.1 实验目的

证明当前无 DP 基线并非“既高精度又安全”，而是“高精度但存在隐私泄漏风险”，为后续固定 DP 和自适应 DP 实验建立安全动机。

### 7.2 论文中要说明的关键观点

这里的表述要尽量严谨，不建议写成“基线方法绝对不安全”，而建议写成：

- 无 DP 基线不具备形式化差分隐私保护
- 在服务器可观察客户端上传模型或更新的条件下，存在可利用的隐私推断风险

### 7.3 推荐攻击方法

主攻击方法建议使用：

- `成员推断攻击（Membership Inference Attack, MIA）`

攻击者身份：

- 联邦服务器

攻击者可见信息：

- 目标客户端上传的本地模型
- 或目标客户端上传的本地更新

攻击目标：

- 判断某个样本是否属于目标客户端的本地训练集

### 7.4 攻击流程设计

建议使用“服务器端白盒成员推断”：

1. 选定一个目标客户端
2. 从该客户端训练集中抽取成员样本
3. 从其他客户端或未参与该客户端训练的样本中抽取非成员样本
4. 获取目标客户端某一轮上传的本地模型
5. 对每个样本计算攻击特征：
   - 交叉熵损失 `loss`
   - 最大类别置信度 `max confidence`
   - 预测熵 `entropy`
   - 真实标签对应概率 `true-label confidence`
6. 使用阈值法或逻辑回归做二分类，判断成员 / 非成员

推荐默认方案：

- 先用 `loss` 阈值法
- 再用逻辑回归融合 4 个特征，作为增强版结果

### 7.5 实验设置

建议优先做：

- 数据集：`MNIST`
- 场景：`Non-IID(alpha=0.1)`
- 算法：至少 `FedAvg`

建议补充：

- `FedProx`
- `FedNova`
- `IID` 场景对照

### 7.6 需要输出的结果

- `attack accuracy`
- `attack AUC`
- `precision / recall / F1`
- ROC 曲线
- 成员与非成员的 `loss` / `confidence` 分布图

### 7.7 预期结论

- 无 DP 下攻击效果应明显高于随机猜测
- Non-IID 场景下泄漏风险可能更明显
- 异构优化算法并不等于隐私保护算法

### 7.8 论文中可支撑的结论

- 后续 DP 的精度下降不是无意义代价
- 它换来的是“攻击成功率下降”这一真实安全收益

## 8. 实验 4：固定差分隐私实验

### 8.1 实验目的

分析固定 DP 噪声带来的“隐私-效能”权衡，建立 DP 基线。

### 8.2 实验设置

建议基于较稳健的算法，例如：

- `FedProx + fixed DP`
- `FedNova + fixed DP`

建议隐私预算：

- `epsilon = 10`
- `epsilon = 5`
- `epsilon = 1`

### 8.3 需要输出的结果

- 不同 `epsilon` 下的最终准确率表
- 不同 `epsilon` 下的收敛曲线
- `epsilon - accuracy` 曲线
- `epsilon - attack AUC` 曲线

### 8.4 预期结论

- 隐私保护越强，精度通常越低
- 攻击成功率下降，但精度也下降
- 固定噪声机制存在明显冲突

### 8.5 可支撑的论文结论

- 固定 DP 不能很好适应训练全过程
- 固定 DP 很难同时兼顾精度和隐私

## 9. 实验 5：自适应 DP 与固定 DP 对比实验

### 9.1 实验目的

证明自适应 DP 比固定 DP 更适合本课题问题。

### 9.2 实验设置

建议比较：

- 无 DP
- 固定 DP
- 自适应 DP

如果最终方案是“异构优化 + 自适应 DP”，则建议固定优化算法，只替换隐私机制。

### 9.3 需要输出的结果

- 最终准确率表
- 每轮精度曲线
- 每轮损失曲线
- 累积隐私预算曲线
- `accuracy - epsilon` 曲线
- `accuracy - attack AUC` 曲线

### 9.4 预期结论

- 自适应 DP 在相近 `epsilon` 下应取得更高准确率
- 或在相近精度下应实现更低攻击成功率
- 自适应机制更能平衡训练早期和训练后期的保护强度

## 10. 实验 6：通信压缩实验

### 10.1 实验目的

验证通信压缩是否能在较小性能损失下减少通信开销。

### 10.2 实验设置

建议对比：

- 无压缩
- `Top-k`

建议参数：

- `topk_ratio = 0.5`
- `topk_ratio = 0.2`
- `topk_ratio = 0.1`

### 10.3 需要输出的结果

- 总通信负载对比表
- 单轮上传 payload 对比表
- `topk_ratio - accuracy` 曲线
- `topk_ratio - 收敛速度` 曲线

### 10.4 预期结论

- 压缩显著降低通信负担
- 压缩过强会损伤性能
- 合理压缩比可取得平衡

## 11. 实验 7：最终方法消融实验

### 11.1 实验目的

分析最终方法各模块的边际贡献。

### 11.2 实验设置

若最终方案为：

- `异构优化 + 自适应 DP + 通信压缩`

建议比较：

- 完整方法
- 去掉自适应 DP
- 去掉异构优化
- 去掉通信压缩
- 只保留基础联邦学习

### 11.3 需要输出的结果

- 消融结果表
- 各版本收敛曲线
- `准确率 / 隐私预算 / 通信开销` 三指标对比

### 11.4 预期结论

- 每个模块都应带来可解释收益
- 异构优化主要改善稳定性
- 自适应 DP 主要改善隐私与精度平衡
- 通信压缩主要减少通信开销

## 15. 建议的论文实验章节结构

建议按下面顺序写：

1. 实验环境与参数设置
2. 对照方法说明
3. 基线算法结果
4. Non-IID 影响分析
5. 安全性论证实验

- 先证明问题存在
- 再证明无 DP 基线不安全
- 再说明固定 DP 的代价
- 最后证明自适应 DP 的优势
