# 毕业设计实现拆解与项目结构规划

## 1. 文档目标

本文档用于将《面向非独立同分布数据的自适应差分隐私联邦学习方法研究》开题报告转化为可执行的工程实施方案，服务后续敏捷开发、实验复现、论文写作与阶段验收。

项目默认定位为研究型联邦学习仿真平台，技术栈采用 `Python + PyTorch + Flower`，围绕以下主线展开：

- 构建 `MNIST` 与 `CIFAR-10` 的 Non-IID 联邦数据仿真环境
- 复现并统一管理 `FedAvg`、`FedProx`、`FedNova` 等基线算法
- 设计并实现自适应差分隐私机制
- 集成通信压缩模块，降低联邦训练通信开销
- 建立对比实验、消融实验和论文制图输出链路

## 2. 课题目标与验收标准

### 2.1 总体目标

完成一个可复现、可扩展、可评估的联邦学习研究代码仓库，支持在 Non-IID 场景下系统分析隐私保护、模型精度、收敛稳定性与通信开销之间的关系，并为毕业论文提供实验数据和图表支撑。

### 2.2 核心研究问题映射

| 开题报告问题 | 工程落地目标 |
| --- | --- |
| Non-IID 导致模型漂移与收敛困难 | 构建异构数据划分模块，接入 FedAvg、FedProx、FedNova 和自定义加权聚合 |
| 固定噪声难平衡隐私与效能 | 设计按训练轮次和梯度统计量自适应调节的 DP 模块 |
| 通信开销高、边缘环境受限 | 实现 Top-k 稀疏化等压缩策略并记录 payload 指标 |

### 2.3 必须完成的交付物

- 一套结构清晰的联邦学习研究代码仓库
- 一组统一配置的基线与改进方法实验脚本
- 一份标准化实验结果输出格式
- 一套对比实验、消融实验与论文图表生成流程
- 一份可用于论文写作和中期检查的实验记录体系

### 2.4 最低验收标准

- `MNIST` 与 `CIFAR-10` 支持 IID 与 Dirichlet Non-IID 划分
- `FedAvg`、`FedProx`、`FedNova` 至少能在同一训练框架中运行
- 差分隐私模块支持梯度裁剪、噪声注入与累计隐私预算记录
- 通信压缩模块支持 Top-k 梯度筛选
- 实验输出至少包含准确率、损失、通信轮次、累计隐私预算、上传 payload
- 可生成对比表与“隐私-效能”曲线

## 3. 项目范围与默认边界

### 3.1 本阶段纳入范围

- 联邦学习仿真，不涉及真实多机部署
- 图像分类任务，首轮数据集限定为 `MNIST` 与 `CIFAR-10`
- 优先支持同步联邦训练流程
- 优先支持研究复现与结果稳定输出，不优先做界面系统

### 3.2 本阶段不纳入范围

- 真实医院、金融或农业业务数据接入
- 复杂密码学协议落地，如安全多方计算或同态加密
- 跨云或跨集群分布式调度
- Web 平台、可视化前端或在线服务化部署

## 5. 统一接口与配置约定

为避免后续实验越做越散，项目必须从第一阶段开始约束统一接口。

### 5.1 统一配置入口

建议所有实验均通过配置文件或统一命令行参数驱动，最少包含以下字段：

| 字段 | 说明 |
| --- | --- |
| `dataset` | 数据集名称，如 `mnist`、`cifar10` |
| `num_clients` | 客户端数量 |
| `dirichlet_alpha` | Non-IID 程度控制参数 |
| `algorithm` | 联邦算法，如 `fedavg`、`fedprox`、`fednova` |
| `dp_mode` | 隐私模式，如 `none`、`fixed`、`adaptive` |
| `epsilon` | 目标隐私预算 |
| `noise_schedule` | 噪声调度策略 |
| `compression_mode` | 压缩模式，如 `none`、`topk` |
| `rounds` | 全局通信轮次 |
| `local_epochs` | 本地训练轮次 |
| `batch_size` | 批大小 |
| `learning_rate` | 学习率 |
| `seed` | 随机种子 |

### 5.2 统一训练接口

建议固化两层执行接口：

- 单次实验运行器：负责加载配置、初始化数据、模型和算法、执行一次完整实验
- 批量实验运行器：负责枚举实验矩阵，批量调用单次运行器，聚合结果

建议统一入口：

- `scripts/run_experiment.py`
- `scripts/run_batch.py`

### 5.3 统一结果落盘格式

每次实验至少生成以下文件：

- `metrics.csv`：每轮准确率、损失、epsilon、payload、round
- `summary.json`：实验配置与最终关键指标
- `train.log`：完整运行日志
- `*.png`：隐私-效能曲线、收敛曲线、通信开销曲线

### 5.4 统一日志字段

每轮至少记录：

- `round`
- `train_loss`
- `test_loss`
- `test_accuracy`
- `epsilon_spent`
- `upload_payload_bytes`
- `selected_clients`
- `algorithm`
- `dp_mode`
- `compression_mode`

## 6. 实施阶段拆解

建议采用 6 个里程碑推进，每个里程碑都应形成可演示成果。

### M1 基础设施与工程初始化

目标：

- 建立可维护的研究代码框架
- 固化依赖、配置、日志、输出规范

核心任务：

- 确定依赖清单与环境安装说明
- 初始化 `src/`、`configs/`、`scripts/`、`tests/`、`outputs/` 目录
- 设计统一配置读取机制
- 设计统一日志与结果输出格式
- 建立 `README` 与实验运行说明

里程碑验收：

- 仓库目录清晰
- 能通过一个最小样例读取配置并写出日志文件

### M2 异构数据仿真环境

目标：

- 构建可控的联邦数据划分环境

核心任务：

- 接入 `MNIST` 与 `CIFAR-10`
- 实现 IID 划分
- 实现 Dirichlet Non-IID 划分
- 支持不同客户端数量和不同 `alpha`
- 输出客户端样本数与标签分布统计

里程碑验收：

- 能针对不同 `alpha` 生成明显不同的客户端标签分布
- 总样本数守恒且每个客户端均可构建 DataLoader

### M3 联邦基线算法

目标：

- 先跑通稳定的联邦训练主链路

核心任务：

- 实现 `FedAvg`
- 接入 `FedProx`
- 接入 `FedNova`
- 统一本地训练、客户端更新、服务器聚合接口
- 在同一数据划分下对比三类算法输出

里程碑验收：

- 三种算法均可运行
- 在同一配置下能输出统一格式结果

### M4 自适应差分隐私机制

目标：

- 在联邦训练中引入可调可追踪的 DP 机制

核心任务：

- 实现梯度裁剪
- 实现固定噪声基线
- 实现按训练轮次调节的噪声策略
- 实现按梯度统计量调节的噪声策略
- 实现累计隐私预算记录与追踪

里程碑验收：

- 支持 `none / fixed / adaptive` 三种隐私模式
- 每轮可输出噪声强度与累计隐私预算

### M5 通信压缩模块

目标：

- 降低联邦训练过程中的通信负载

核心任务：

- 实现 Top-k 稀疏化
- 记录压缩前后 payload
- 验证与 DP 模块兼容
- 验证与 `FedProx`、`FedNova` 兼容

里程碑验收：

- 压缩模块可独立启用或关闭
- 能输出压缩率与 payload 变化

### M6 实验体系与论文支撑

目标：

- 形成完整的研究验证闭环

核心任务：

- 设计实验矩阵
- 跑通对比实验
- 跑通消融实验
- 生成精度、收敛、通信、隐私预算相关图表
- 生成“隐私-效能”权衡曲线
- 整理论文所需表格与结论摘要

里程碑验收：

- 可生成论文可直接使用的表图结果
- 实验记录可追踪、可重复

## 8. 实验设计建议

### 8.1 首轮对比实验矩阵

建议首轮实验方法包括：

- `FedAvg`
- `FedProx`
- `FedNova`
- `DP-FedAvg`（固定噪声）
- 自定义方法（自适应 DP + 异构优化 + 通信压缩）

### 8.2 控制变量

重点控制以下因素：

- 数据集：`MNIST`、`CIFAR-10`
- 客户端数量：如 `10`、`50`
- Non-IID 程度：如 `alpha=1.0`、`0.5`、`0.1`
- 隐私模式：无 DP、固定 DP、自适应 DP
- 压缩模式：无压缩、Top-k

### 8.3 核心指标

- 测试准确率
- 测试损失
- 收敛轮次
- 累计隐私预算 `epsilon`
- 单轮上传 payload
- 总通信载荷

### 8.4 消融实验建议

围绕最终方法逐项移除组件：

- 去掉自适应噪声，仅保留固定噪声
- 去掉异构优化，仅保留基础聚合
- 去掉压缩模块，仅保留完整上传

目标：

- 分析每个模块对精度、收敛、隐私预算和通信开销的边际贡献

## 9. 测试与验收方案

### 9.1 单元测试

- 数据划分正确性测试
- 梯度裁剪边界测试
- 噪声注入输出维度和数值稳定性测试
- Top-k 稀疏化保留比例测试
- 聚合逻辑输入输出一致性测试

### 9.2 集成测试

- `FedAvg` 在 IID 数据上的最小联邦训练测试
- `FedProx` 在 Non-IID 数据上的运行测试
- `DP + FedProx` 组合测试
- `DP + FedNova` 组合测试
- `DP + Top-k` 组合测试

### 9.3 文档中必须保留的验收场景

- 不同 `alpha` 下客户端标签分布确实变化，且总样本数守恒
- `FedAvg`、`FedProx`、`FedNova` 在同一配置下可重复运行并输出稳定日志
- 噪声注入、梯度裁剪、隐私预算累计结果可追踪
- `DP + FedProx`、`DP + FedNova`、`DP + Top-k` 组合运行不报错
- 每次实验都能产出精度、损失、通信开销、隐私预算四类核心指标
- 能够生成对比表和“隐私-效能”曲线，满足论文写作取数需要

## 11. 推荐实施顺序

为了控制风险，推荐严格按以下顺序推进：

1. 先做最小可运行联邦基线，不先做复杂 DP
2. 再做可控的 Non-IID 数据划分与结果统计
3. 在稳定基线上加入 `FedProx` 和 `FedNova`
4. 先实现固定噪声 DP，再迭代到自适应 DP
5. 在算法稳定后再加通信压缩
6. 最后再做批量实验、图表导出与论文材料整理

这样可以避免同时引入多个变量，便于调试、定位问题与写论文时做因果解释。

## 12. 风险点与规避建议

### 12.1 精度波动大

原因：

- Non-IID 过强
- DP 噪声过大
- 压缩比例过高

建议：

- 先在 `MNIST` 上调通
- 优先建立无 DP、无压缩基线
- 逐模块增量启用

### 12.2 结果难复现

原因：

- 随机种子不统一
- 配置分散
- 输出格式不固定

建议：

- 强制记录随机种子
- 所有实验必须保存完整配置
- 所有结果统一落盘格式

### 12.3 模块耦合过深

原因：

- DP、压缩、聚合逻辑交叉写死

建议：

- 训练、隐私、压缩、聚合分别模块化
- 所有策略通过配置组合，而不是在主流程里硬编码
