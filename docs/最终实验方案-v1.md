# 最终实验方案 v1

## 1. 文档目的

这份文档用于冻结当前阶段的“最终实验方案 v1”，作为后续实验执行、结果整理和论文写作的统一依据。

它和 [毕业论文实验设计方案.md](/Users/admin/Desktop/fl-adaptive-dp/docs/毕业论文实验设计方案.md) 的区别是：

- 那份文档更偏“完整蓝图”
- 这份文档更偏“当前可执行版本”

换句话说，这份文档回答的是：

- 现在到底先做哪些实验
- 每组实验要产出哪些素材
- 这些素材将来分别写进论文哪里

## 2. 当前项目状态判断

截至目前，项目已经具备以下基础：

- `MNIST / CIFAR-10` 数据集接入
- IID / Dirichlet Non-IID 划分
- `FedAvg / FedProx / FedNova` 三条联邦训练链路
- 固定噪声 DP
- 可审计自适应 DP
- `Top-k` 通信压缩
- 统一配置、日志和实验输出目录

因此，后续重点不再是继续扩展大量新模块，而是：

1. 把论文的主问题逐个用实验讲清楚
2. 把实验结果整理成可直接写进论文的表格和图
3. 通过多随机种子和第二数据集提升结论可信度

## 2.1 本版方案新增的约束

为避免后续实验“能跑但不能公平比较”，本版方案在原有主线基础上新增 3 条约束：

1. 差分隐私实验采用统一口径，保证 `fixed DP` 与 `adaptive DP` 的结果可直接比较。
2. 安全性实验采用统一威胁模型，保证攻击结果可复现、可解释。
3. DP 主线实验统一选择 1 条优化底座，避免在三种方法上重复投入过多工作量。

## 3. 论文要最终回答的核心问题

整篇论文最终建议围绕以下 6 个问题组织实验：

1. `Non-IID` 是否显著影响联邦学习性能。
2. `FedProx / FedNova` 是否能缓解 `Non-IID` 带来的退化。
3. 无 DP 基线是否存在可利用的隐私泄漏风险。
4. 固定 DP 是否带来明显的“隐私-效能”冲突。
5. 自适应 DP 是否优于固定 DP。
6. 通信压缩与最终组合方法是否在精度、隐私和通信之间取得更好平衡。

这 6 个问题里，前 5 个是论文主线，第 6 个是最终方案落地价值。

## 3.1 统一实验原则

### 原则 1：基线实验与 DP 主线实验分层

- `FedAvg / FedProx / FedNova` 三种方法都用于基线层分析
- `DP` 主线实验只固定 1 条优化底座完整展开
- 若时间允许，再用另一条方法做小规模补充验证

建议落地方式：

- 实验 `A / B`：三种方法都做
- 实验 `C`：至少做 `FedAvg`，最好再补“最终选定底座”
- 实验 `D / E / F / G`：统一使用 1 条优化底座

### 原则 2：DP 口径统一

所有 `DP` 相关实验必须使用同一套隐私定义与统计规则，至少统一以下内容：

- 隐私定义：全文统一写为 `(\epsilon, \delta)-DP`
- `delta` 取值规则：全文固定，不随实验单独修改
- 裁剪对象：统一为“客户端更新”或“梯度”，不可混用
- 裁剪方式：统一采用同一种范数裁剪方式
- `clip` 含义：统一指同一个裁剪阈值
- `accountant`：全文统一使用同一种隐私会计方法
- 采样率定义：统一按同一种客户端采样策略计算

这里的核心不是“现在立刻把所有最优参数锁死”，而是先把比较规则固定下来。后续可以在同一规则下调参，但不能一组实验一个定义。

### 原则 3：固定 DP 与自适应 DP 的公平比较

比较 `fixed DP` 与 `adaptive DP` 时，必须满足：

- 使用同一优化底座
- 使用同一数据划分
- 使用同一模型结构
- 使用同一裁剪机制
- 使用同一 `delta`
- 使用同一 `accountant`

最终比较建议采用两种口径二选一或同时给出：

- 相近累计隐私预算下比较精度
- 相近精度水平下比较攻击效果

### 原则 4：正式实验与预实验分离

为避免过早锁死参数，本方案将参数分为两类：

- 预实验可调参数：`lr`、`mu`、`clip`、自适应噪声调度范围、`topk_ratio`
- 正式实验冻结参数：一旦进入正式对比实验，除被研究变量外其余参数全部保持不变

建议流程：

1. 先用小规模预实验确定稳定参数区间
2. 再冻结正式实验配置
3. 正式结果只采用冻结后的配置

## 4. 最终实验主线

## 4.1 实验 A：基线算法对比实验

### 目标

建立无隐私基线，回答：

- `FedAvg / FedProx / FedNova` 在 IID 与 Non-IID 下的差异是什么
- 哪条基线更适合作为后续 DP 实验底座

### 方法

- `FedAvg`
- `FedProx`
- `FedNova`

### 数据集

- `MNIST`
- `CIFAR-10`

### 数据分布

- IID
- Dirichlet Non-IID

建议：

- `MNIST` 用 `alpha = 0.1`
- `CIFAR-10` 优先也用 `alpha = 0.1`

### 参数建议

先沿用当前较优设置，再视 `CIFAR-10` 收敛情况微调：

- `num_clients = 10`
- `rounds = 20`
- `local_epochs = 1`
- `batch_size = 64`
- `fraction_fit = 1.0`

补充建议：

- `rounds = 20` 可作为快速验证轮数
- 若 `CIFAR-10` 在 `Non-IID` 或 `DP` 条件下收敛不足，正式实验应适当增加训练轮数
- 正式版中应区分“预实验轮数”和“正式实验轮数”，避免直接混用

当前 `MNIST` 可参考：

- `FedAvg`: `lr = 0.05`
- `FedProx`: `lr = 0.05, mu = 0.5`
- `FedNova`: `lr = 0.05`

### 需要产出

表格：

- 基线方法最终准确率表
- 基线方法最终损失表
- IID 到 Non-IID 的退化幅度表
- 稳定性指标表

图像：

- IID 准确率曲线
- IID 损失曲线
- Non-IID 准确率曲线
- Non-IID 损失曲线

### 论文可支撑的结论

- IID 下三种方法差距不大
- Non-IID 下 `FedAvg` 更容易退化
- `FedProx / FedNova` 更适合作为异构环境下的优化基线

### 本实验的额外任务

除输出结果外，还要完成 1 个选择：

- 在 `FedProx` 与 `FedNova` 中选出后续 `DP` 主线实验的统一底座

建议选择标准：

- 若更强调稳定性与解释性，优先 `FedProx`
- 若更强调最终精度，优先 `FedNova`
- 一旦确定，后续实验 `D / E / F / G` 全文保持一致

### 当前状态

- `MNIST` 已完成正式版
- `CIFAR-10` 还需要补

系统回答：

- 数据异构程度究竟如何影响性能
- 哪种方法对异构更敏感

### 方法

- `FedAvg`
- `FedProx`
- `FedNova`

### 数据集

- `MNIST`
- 若时间允许，再补 `CIFAR-10`

### 数据分布

### 需要产出

## 4.3 实验 C：安全性论证实验

### 目标

证明无 DP 基线并不是“高精度且安全”，而是“高精度但存在隐私泄漏风险”。

### 攻击方法

建议采用：

- `Membership Inference Attack (MIA)`

攻击者：

- 联邦服务器

可见信息：

- 建议统一为：客户端上传的本地更新

### 威胁模型固定建议

为保证安全性实验能稳定复现，建议在正式实验前固定以下协议：

- 攻击者身份：诚实但好奇的服务器
- 攻击目标：判断样本是否参与某一客户端的本地训练
- 攻击输入：客户端单轮上传更新，或固定若干轮的更新统计量
- 攻击输出：成员 / 非成员二分类结果
- 攻击评估：`AUC` 作为主指标，`Accuracy / Precision / Recall / F1` 作为辅助指标

建议优先采用单一、可解释的攻击方案，不要同时混入过多攻击变体。

### 方法

建议至少做：

- `FedAvg`

建议补充：

- “最终选定的 DP 底座方法”

### 数据集

- `MNIST`

### 数据分布

优先：

- `Non-IID(alpha = 0.1)`

补充：

- IID

### 需要产出

表格：

- 攻击准确率表
- AUC / Precision / Recall / F1 表

图像：

- ROC 曲线
- 成员 / 非成员损失分布图
- 成员 / 非成员置信度分布图

### 论文可支撑的结论

- 无 DP 基线不具备形式化隐私保护
- 后续 DP 的精度下降换来真实隐私收益

### 执行注意事项

- 成员样本与非成员样本的构造方式必须全文一致
- 攻击训练集、验证集、测试集划分要固定随机种子
- 若最终只保留一种攻击方法，正文里要解释“为何选它而不是做很多攻击”

## 4.4 实验 D：固定 DP 实验

### 目标

建立“固定差分隐私”基线，回答：

- 隐私保护增强后，精度下降了多少
- 攻击效果下降了多少

### 方法

建议选 1 条较稳的优化方法作为底座：

- `FedProx + fixed DP`

或

- `FedNova + fixed DP`

若时间允许，两条都做更好。

正式版建议：

- 主实验只保留 1 条统一底座
- 另一条方法最多做补充验证，不进入主表主图

### 数据集

- `MNIST`
- 后续扩展 `CIFAR-10`

### 数据分布

- `Non-IID(alpha = 0.1)`

### 参数建议

建议至少比较：

- `epsilon = 10`
- `epsilon = 5`
- `epsilon = 1`

同时应在正式实验前统一冻结：

- `delta`
- `clip`
- 裁剪对象与裁剪方式
- `accountant`
- 客户端采样率计算方式

推荐执行方式：

- 先通过预实验确定 `clip` 的稳定区间
- 再在固定 `clip`、固定 `delta`、固定 `accountant` 下比较不同 `epsilon`
- 论文正文中不能出现“同为 `epsilon = 5` 但记账规则不同”的情况

### 需要产出

表格：

- 不同 `epsilon` 下准确率表
- 不同 `epsilon` 下攻击指标表

图像：

- `epsilon - accuracy` 曲线
- `epsilon - attack AUC` 曲线
- 不同 `epsilon` 的收敛曲线

### 论文可支撑的结论

- 固定 DP 存在明显“隐私-效能”冲突
- 噪声越大，攻击越难，但精度也越差

## 4.5 实验 E：自适应 DP 与固定 DP 对比实验

### 目标

回答论文最核心的问题：

- 自适应 DP 是否比固定 DP 更优

### 方法

建议统一优化底座，例如：

- `FedProx + no DP`
- `FedProx + fixed DP`
- `FedProx + adaptive DP`

或者改用 `FedNova` 作为统一底座，但必须全文一致。

正式比较时建议固定以下对照组：

- `base + no DP`
- `base + fixed DP`
- `base + adaptive DP`

其中 `base` 为实验 A 中选定的统一优化底座。

### 数据集

- `MNIST`
- 后续扩展 `CIFAR-10`

### 数据分布

- `Non-IID(alpha = 0.1)`

### 需要产出

表格：

- 最终准确率对比表
- 最终攻击效果对比表
- 最终隐私预算对比表

图像：

- 收敛曲线
- 累积 `epsilon` 曲线
- `accuracy - epsilon` 曲线
- `accuracy - attack AUC` 曲线

正文中应明确说明：

- `adaptive DP` 改变的是噪声调度策略，而不是更换整套隐私定义
- 所有预算均由同一 `accountant` 累积得到
- 与 `fixed DP` 对比时，其余设置保持不变

### 论文可支撑的结论

- 自适应 DP 在相近预算下精度更高
- 或在相近精度下攻击更弱
- 自适应机制比固定噪声更符合联邦训练过程

## 4.6 实验 F：通信压缩实验

### 目标

回答：

- `Top-k` 是否能明显减少通信开销
- 它会带来多大精度损失

### 方法

建议比较：

- 无压缩
- `Top-k(0.5)`
- `Top-k(0.2)`
- `Top-k(0.1)`

额外建议：

- 在正式实验前明确 `Top-k` 与 `DP` 的组合顺序
- 全文固定采用同一种顺序，避免不同实验流程不一致

建议优先采用：

- 本地更新裁剪
- 加噪
- 再做压缩与上传

如果当前实现不是这一顺序，也可以保持现有实现，但必须全文统一并在论文中写清楚原因。

### 数据集

- `MNIST`

### 数据分布

- `Non-IID(alpha = 0.1)`

### 需要产出

表格：

- 总通信开销表
- 单轮平均通信开销表
- 压缩率与最终准确率表

图像：

- `topk_ratio - accuracy` 曲线
- `topk_ratio - upload_payload` 曲线

### 论文可支撑的结论

- 合理压缩比可以明显降低通信负担
- 压缩过强会带来明显性能损失

## 4.7 实验 G：最终方法与消融实验

### 目标

证明最终方法的改进不是偶然，而是多个模块共同作用的结果。

### 最终方法建议定义

建议统一为：

- `异构优化 + 自适应 DP + Top-k`

这里“异构优化”建议二选一并固定：

- `FedProx`

或

- `FedNova`

从当前结果看，如果强调最终精度，可优先考虑 `FedNova`；如果强调稳定性，可优先考虑 `FedProx`。正文里要保持前后一致。

推荐写法：

- 若正文主线偏“稳健训练”，最终方法写成 `FedProx + adaptive DP + Top-k`
- 若正文主线偏“最终精度”，最终方法写成 `FedNova + adaptive DP + Top-k`

二者选其一，不建议在最终方法定义中来回切换。

### 消融版本

- 基础联邦学习
- `异构优化`
- `异构优化 + fixed DP`
- `异构优化 + adaptive DP`
- `异构优化 + adaptive DP + Top-k`

### 需要产出

表格：

- 消融结果表
- 三指标对比表：准确率 / 隐私预算 / 通信开销

图像：

- 消融柱状图
- 最终方法与对照方法曲线图

### 论文可支撑的结论

- 每个模块都带来可解释收益
- 最终方案在多个目标之间更平衡

如果按论文推进效率来讲，现在最应该优先推进的是：

1. `CIFAR-10` 上的基线正式版实验
