# 最终实验方案 v1

## 1. 文档目的

这份文档用于冻结当前阶段的“最终实验方案 v1”，作为后续实验执行、结果整理和论文写作的统一依据。

它和 [毕业论文实验设计方案.md](/Users/admin/Desktop/fl-adaptive-dp/docs/毕业论文实验设计方案.md) 的区别是：

- 那份文档更偏“完整蓝图”
- 这份文档更偏“当前可执行版本”

换句话说，这份文档回答的是：

- 现在到底先做哪些实验
- 每组实验要产出哪些素材
- 这些素材将来分别写进论文哪里

## 2. 当前项目状态判断

截至目前，项目已经具备以下基础：

- `MNIST / CIFAR-10` 数据集接入
- IID / Dirichlet Non-IID 划分
- `FedAvg / FedProx / FedNova` 三条联邦训练链路
- 固定噪声 DP
- 可审计自适应 DP
- `Top-k` 通信压缩
- 统一配置、日志和实验输出目录

因此，后续重点不再是继续扩展大量新模块，而是：

1. 把论文的主问题逐个用实验讲清楚
2. 把实验结果整理成可直接写进论文的表格和图
3. 通过多随机种子和第二数据集提升结论可信度

## 3. 论文要最终回答的核心问题

整篇论文最终建议围绕以下 6 个问题组织实验：

1. `Non-IID` 是否显著影响联邦学习性能。
2. `FedProx / FedNova` 是否能缓解 `Non-IID` 带来的退化。
3. 无 DP 基线是否存在可利用的隐私泄漏风险。
4. 固定 DP 是否带来明显的“隐私-效能”冲突。
5. 自适应 DP 是否优于固定 DP。
6. 通信压缩与最终组合方法是否在精度、隐私和通信之间取得更好平衡。

这 6 个问题里，前 5 个是论文主线，第 6 个是最终方案落地价值。

## 4. 最终实验主线

## 4.1 实验 A：基线算法对比实验

### 目标

建立无隐私基线，回答：

- `FedAvg / FedProx / FedNova` 在 IID 与 Non-IID 下的差异是什么
- 哪条基线更适合作为后续 DP 实验底座

### 方法

- `FedAvg`
- `FedProx`
- `FedNova`

### 数据集

- `MNIST`
- `CIFAR-10`

### 数据分布

- IID
- Dirichlet Non-IID

建议：

- `MNIST` 用 `alpha = 0.1`
- `CIFAR-10` 优先也用 `alpha = 0.1`

### 参数建议

先沿用当前较优设置，再视 `CIFAR-10` 收敛情况微调：

- `num_clients = 10`
- `rounds = 20`
- `local_epochs = 1`
- `batch_size = 64`
- `fraction_fit = 1.0`

当前 `MNIST` 可参考：

- `FedAvg`: `lr = 0.05`
- `FedProx`: `lr = 0.05, mu = 0.5`
- `FedNova`: `lr = 0.05`

### 需要产出

表格：

- 基线方法最终准确率表
- 基线方法最终损失表
- IID 到 Non-IID 的退化幅度表
- 稳定性指标表

图像：

- IID 准确率曲线
- IID 损失曲线
- Non-IID 准确率曲线
- Non-IID 损失曲线

### 论文可支撑的结论

- IID 下三种方法差距不大
- Non-IID 下 `FedAvg` 更容易退化
- `FedProx / FedNova` 更适合作为异构环境下的优化基线

### 当前状态

- `MNIST` 已完成正式版
- `CIFAR-10` 还需要补

## 4.2 实验 B：Non-IID 强度影响实验

### 目标

系统回答：

- 数据异构程度究竟如何影响性能
- 哪种方法对异构更敏感

### 方法

- `FedAvg`
- `FedProx`
- `FedNova`

### 数据集

- `MNIST`
- 若时间允许，再补 `CIFAR-10`

### 数据分布

- `alpha = 1.0`
- `alpha = 0.5`
- `alpha = 0.1`

### 需要产出

表格：

- 不同 `alpha` 下最终准确率表
- 不同 `alpha` 下最终损失表

图像：

- `alpha - accuracy` 曲线
- `alpha - loss` 曲线
- `alpha - 收敛速度` 曲线

### 论文可支撑的结论

- `alpha` 越小，异构越强，性能整体越差
- `FedAvg` 对异构更敏感
- `FedProx / FedNova` 退化更缓

## 4.3 实验 C：安全性论证实验

### 目标

证明无 DP 基线并不是“高精度且安全”，而是“高精度但存在隐私泄漏风险”。

### 攻击方法

建议采用：

- `Membership Inference Attack (MIA)`

攻击者：

- 联邦服务器

可见信息：

- 客户端上传的本地模型
- 或客户端上传的本地更新

### 方法

建议至少做：

- `FedAvg`

建议补充：

- `FedProx`
- `FedNova`

### 数据集

- `MNIST`

### 数据分布

优先：

- `Non-IID(alpha = 0.1)`

补充：

- IID

### 需要产出

表格：

- 攻击准确率表
- AUC / Precision / Recall / F1 表

图像：

- ROC 曲线
- 成员 / 非成员损失分布图
- 成员 / 非成员置信度分布图

### 论文可支撑的结论

- 无 DP 基线不具备形式化隐私保护
- 后续 DP 的精度下降换来真实隐私收益

## 4.4 实验 D：固定 DP 实验

### 目标

建立“固定差分隐私”基线，回答：

- 隐私保护增强后，精度下降了多少
- 攻击效果下降了多少

### 方法

建议选 1 条较稳的优化方法作为底座：

- `FedProx + fixed DP`

或

- `FedNova + fixed DP`

若时间允许，两条都做更好。

### 数据集

- `MNIST`
- 后续扩展 `CIFAR-10`

### 数据分布

- `Non-IID(alpha = 0.1)`

### 参数建议

建议至少比较：

- `epsilon = 10`
- `epsilon = 5`
- `epsilon = 1`

### 需要产出

表格：

- 不同 `epsilon` 下准确率表
- 不同 `epsilon` 下攻击指标表

图像：

- `epsilon - accuracy` 曲线
- `epsilon - attack AUC` 曲线
- 不同 `epsilon` 的收敛曲线

### 论文可支撑的结论

- 固定 DP 存在明显“隐私-效能”冲突
- 噪声越大，攻击越难，但精度也越差

## 4.5 实验 E：自适应 DP 与固定 DP 对比实验

### 目标

回答论文最核心的问题：

- 自适应 DP 是否比固定 DP 更优

### 方法

建议统一优化底座，例如：

- `FedProx + no DP`
- `FedProx + fixed DP`
- `FedProx + adaptive DP`

或者改用 `FedNova` 作为统一底座，但必须全文一致。

### 数据集

- `MNIST`
- 后续扩展 `CIFAR-10`

### 数据分布

- `Non-IID(alpha = 0.1)`

### 需要产出

表格：

- 最终准确率对比表
- 最终攻击效果对比表
- 最终隐私预算对比表

图像：

- 收敛曲线
- 累积 `epsilon` 曲线
- `accuracy - epsilon` 曲线
- `accuracy - attack AUC` 曲线

### 论文可支撑的结论

- 自适应 DP 在相近预算下精度更高
- 或在相近精度下攻击更弱
- 自适应机制比固定噪声更符合联邦训练过程

## 4.6 实验 F：通信压缩实验

### 目标

回答：

- `Top-k` 是否能明显减少通信开销
- 它会带来多大精度损失

### 方法

建议比较：

- 无压缩
- `Top-k(0.5)`
- `Top-k(0.2)`
- `Top-k(0.1)`

### 数据集

- `MNIST`

### 数据分布

- `Non-IID(alpha = 0.1)`

### 需要产出

表格：

- 总通信开销表
- 单轮平均通信开销表
- 压缩率与最终准确率表

图像：

- `topk_ratio - accuracy` 曲线
- `topk_ratio - upload_payload` 曲线

### 论文可支撑的结论

- 合理压缩比可以明显降低通信负担
- 压缩过强会带来明显性能损失

## 4.7 实验 G：最终方法与消融实验

### 目标

证明最终方法的改进不是偶然，而是多个模块共同作用的结果。

### 最终方法建议定义

建议统一为：

- `异构优化 + 自适应 DP + Top-k`

这里“异构优化”建议二选一并固定：

- `FedProx`

或

- `FedNova`

从当前结果看，如果强调最终精度，可优先考虑 `FedNova`；如果强调稳定性，可优先考虑 `FedProx`。正文里要保持前后一致。

### 消融版本

- 基础联邦学习
- `异构优化`
- `异构优化 + fixed DP`
- `异构优化 + adaptive DP`
- `异构优化 + adaptive DP + Top-k`

### 需要产出

表格：

- 消融结果表
- 三指标对比表：准确率 / 隐私预算 / 通信开销

图像：

- 消融柱状图
- 最终方法与对照方法曲线图

### 论文可支撑的结论

- 每个模块都带来可解释收益
- 最终方案在多个目标之间更平衡
