# 实验 1：FedAvg / FedProx / FedNova 在 IID 与 Non-IID 下的对比实验记录

## 1. 实验目的

本实验用于验证三种联邦优化基线算法在统一设置下的表现差异，重点回答两个问题：

1. 非独立同分布数据是否会显著降低联邦学习性能。
2. `FedProx`、`FedNova` 是否能在当前实现和参数设置下缓解 `FedAvg` 在强 Non-IID 场景中的退化。

这组实验是整篇论文实验部分的第一组基础实验，作用是建立后续固定 DP、自适应 DP 和通信压缩实验的对照基线。

## 2. 参数设置依据

本实验参数不是随意拍脑袋定的，而是参考了联邦学习经典文献和官方实现建议后，再结合当前项目算力与代码成熟度进行缩放。

### 2.1 文献依据

- `FedAvg` 原始论文提出了“少量客户端参与 + 多轮通信 + 本地多步更新”的基本实验范式：
  [Communication-Efficient Learning of Deep Networks from Decentralized Data](https://huggingface.co/papers/1602.05629)
- `FedProx` 官方仓库明确建议：如果数据集和设置不同，`mu` 需要调参，推荐从 `{0.001, 0.01, 0.1, 0.5, 1}` 中选择：
  [litian96/FedProx](https://github.com/litian96/FedProx)
- `FedNova` 论文核心思想是通过对本地更新进行归一化来缓解异构环境下的目标不一致问题：
  [Tackling the Objective Inconsistency Problem in Heterogeneous Federated Optimization](https://arxiv.org/pdf/2007.07481.pdf)

### 2.2 本实验实际采用的设置

考虑到当前项目还处于毕业设计实现阶段，本实验选择“论文思路一致，但规模适度收缩”的配置：

- 数据集：`MNIST`
- 客户端数：`10`
- 每轮参与客户端比例：`0.5`
- 总通信轮数：`10`
- 本地 epoch：`1`
- batch size：`64`
- 学习率：`0.01`
- 随机种子：`42`
- 非 IID 划分：Dirichlet，`alpha = 0.1`
- `FedProx` 近端系数：`mu = 0.01`

### 2.3 为什么先用这套设置

- `MNIST` 足够快，适合先做第一轮论文基线验证
- `alpha = 0.1` 属于较强 Non-IID，更容易观察算法差异
- `mu = 0.01` 是 FedProx 官方建议调参集合中的一个温和值
- `10` 轮通信已经足够观察“是否明显退化”和“是否存在恢复趋势”

## 5. 结果分析

### 5.1 已经可以明确成立的结论

#### 结论 1：强 Non-IID 显著降低联邦学习性能

这一点在当前实验里非常明显。

- `FedAvg` 从 IID 的 `79.95%` 降到 Non-IID 的 `50.20%`
- `FedProx` 从 IID 的 `79.90%` 降到 Non-IID 的 `50.20%`
- `FedNova` 从 IID 的 `79.95%` 降到 Non-IID 的 `48.70%`

这说明数据异构性确实会显著破坏联邦训练效果，可以直接支撑论文里“Non-IID 是联邦学习核心挑战之一”的论断。

#### 结论 2：IID 场景下三种算法差异很小

这也是一个很重要的实验现象：

- 三种算法在 IID 下都稳定在约 `79.9%`
- 说明算法差异不是来源于简单场景下的“模型能力不同”
- 后续如果出现差异，更有可能确实来自对异构性的处理能力不同

这对论文论证是有利的。

### 5.2 当前结果中不能过度宣称的部分

#### 结论 3：在当前参数和轮数下，FedProx 没有明显优于 FedAvg

这是本次实验最需要诚实面对的点。

在当前设置下：

- `FedProx` 最终准确率与 `FedAvg` 几乎相同
- 两者在 Non-IID 下的最好轮次准确率也几乎一致

这意味着你**现在不能直接写出“FedProx 明显优于 FedAvg”** 这样的结论。

更稳妥的表述应该是：

- 在当前实现和超参数设置下，`FedProx` 未表现出显著优势
- 后续需要通过更长训练轮次或 `mu` 调参进一步验证其潜力

#### 结论 4：FedNova 在当前设置下表现出更高峰值，但最终稳定性不足

本次实验里：

- `FedNova` 的 Non-IID 最佳准确率最高，达到 `61.75%`
- 但第 10 轮最终准确率回落到 `48.70%`

这说明：

- `FedNova` 在当前实现中有一定潜力
- 但训练后期稳定性不够，需要进一步检查聚合策略、轮数和超参数

因此当前更合理的结论是：

- `FedNova` 在强 Non-IID 下表现出更高的短期恢复能力
- 但最终轮次表现不稳定，尚不能直接下结论说它全面优于 `FedAvg`

## 6. 论文里应该怎么写这组实验

这组实验最适合写成“基线验证 + 异构性效应证明”，而不是现在就强行写成“某个算法全面领先”。

建议论文里这样表述：

### 6.1 可以直接写的内容

- 在 IID 场景下，`FedAvg`、`FedProx` 与 `FedNova` 取得了接近的精度，说明三者在同质数据条件下性能差异不显著。
- 在强 Non-IID 场景下，三种方法的最终精度均明显下降，说明统计异构性会显著影响联邦训练效果。
- `FedNova` 在中间轮次达到更高峰值精度，表明归一化聚合策略在缓解异构性方面具有一定潜力。

### 6.2 暂时不要写死的内容

- 不要直接写“FedProx 明显优于 FedAvg”
- 不要直接写“FedNova 最优”

因为当前实验结果还不足以支撑这两个强结论。

## 7. 这组实验暴露出的后续工作

这不是坏消息，反而很有价值。它告诉我们，下一步该怎么把实验做得更像一篇论文，而不是一组演示。

### 7.1 需要补做的改进实验

建议紧接着做以下 3 件事：

1. 对 `FedProx` 做 `mu` 调参实验  
   建议至少尝试：
   - `mu = 0.001`
   - `mu = 0.01`
   - `mu = 0.1`

2. 将通信轮数从 `10` 增加到 `20` 或 `30`  
   原因是 `10` 轮更适合快速验证，不一定足够体现异构优化算法的长期差异。

3. 引入多随机种子重复实验  
   建议至少：
   - `seed = 42`
   - `seed = 123`
   - `seed = 2026`

### 7.2 为什么这三步很关键

- `FedProx` 对 `mu` 很敏感，不调参很容易看不出收益
- `FedNova` 的优势更可能体现在更长训练过程中的收敛行为
- 单个随机种子的结果不适合直接写成最终论文结论

## 8. 当前可以支撑的中间结论

如果你现在就要写阶段性汇报、中期检查或者代码开发记录，这组实验已经可以给出以下中间结论：

1. 当前实现已经成功复现了 IID 与 Non-IID 的显著性能差异
2. 当前基线实验为后续隐私模块接入提供了可靠对照组
3. `FedProx` 和 `FedNova` 的潜力需要在更充分调参和更长轮次下继续验证

## 9. 下一步建议

从论文推进效率来看，建议你接下来优先做：

1. `FedProx` 的 `mu` 调参实验
2. `FedAvg / FedProx / FedNova` 在 `20~30` 轮下的复验
3. 多随机种子重复实验

等这三步完成后，再进入固定 DP 实验会更稳，因为那时你的“无隐私基线”会更加可信。
